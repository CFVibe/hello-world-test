## CRITICAL: Telegram Formatting Rule (learned the hard way)
- NEVER use backtick code formatting in Telegram replies (e.g. `suppressToolErrors: true`)
- NEVER use markdown bold, italic, or headers in Telegram replies
- OpenClaw converts markdown to HTML for Telegram — if conversion fails, the ENTIRE message is silently dropped (not even delivered as plain text)
- Write pure plain prose only. Use quotes or CAPS for emphasis if needed.
- Rule from SOUL.md: "Plain text in Telegram. No markdown headers or bold."
- This caused at least two confirmed silent message drops on 2026-02-18

## NotebookLM (how we use it)
- Install NotebookLM CLI + MCP server: uv tool install notebooklm-mcp-cli (provides nlm + notebooklm-mcp)
- Register MCP server with mcporter (stdio): mcporter config add notebooklm --command notebooklm-mcp
- Auth: use nlm login (auto via Chrome CDP if possible). If auth expires/doesn’t stick, use nlm login --manual with the exact cookie header copied from a NotebookLM batchexecute request (DO NOT store cookies in memory).
- Use via MCP through mcporter:
  - mcporter call notebooklm.notebook_list
  - mcporter call notebooklm.notebook_create {title}
  - mcporter call notebooklm.source_add (text/url/file/drive)
  - mcporter call notebooklm.studio_create (artifact_type=infographic, etc.)
  - mcporter call notebooklm.studio_status
  - Download artifacts via notebooklm.download_artifact / nlm download when supported (infographic download currently flaky; fallback is to open the notebook UI and download there)
